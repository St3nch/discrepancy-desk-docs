# Chrome Extension Plan

## Status

Planning only. No extension code exists.

## Purpose

Reduce manual capture friction for **X** after the core X workflow is stable.

The extension is not part of the initial MVP. It is a later helper, admitted only after platform policy review and measured operator need.

## Platform Admission

### X

X is the only admitted DOM-capture target for this plan.

Any X capture implementation must remain:

- explicitly human-triggered per item
- local-only
- visible to the operator
- editable before save
- fail-closed when parsing fails
- free of background collection
- free of timeline, profile, search-result, or account crawling
- free of automated publishing or engagement actions

X policy and API access must be rechecked before implementation. The extension must not use undocumented endpoints or session-cookie calls.

### Truth Social

Truth Social is **not admitted as a browser-extension capture target**.

The accepted research posture is:

```text
URLs, human-authored notes, owned-content records, official share tools,
official bookmarks, manual metrics, and official owner export only.
```

Do not build or ship any Truth Social:

- content script
- DOM parser
- post-text extraction
- author/timestamp extraction
- metrics extraction
- thread capture
- timeline/search/profile capture
- background listener
- automated screenshot feature

This boundary comes from `06-research/truth-social-capture-boundaries.md`. It may change only after written permission, materially revised official rules, or owner-accepted professional legal guidance.

## X Capture Philosophy

Build a deliberately small bookmark-with-context helper, not a scraper.

Candidate behavior:

1. Human navigates to a specific X post.
2. Human clicks the extension.
3. Extension attempts a narrow extraction from the active post only.
4. Extension displays the candidate capture.
5. Human edits or rejects it.
6. Approved content is sent to the local B1-L7 Control Room.

No invisible collection occurs.

## Candidate Capture Envelope

This is a provisional field inventory, not an accepted database schema:

- platform
- page URL
- post ID if reliably detectable
- author handle and display name
- visible post text
- visible timestamp
- selected text fallback
- optional human note
- tags
- capture type
- parser version
- capture timestamp generated by the backend
- review status

Real captures must be inspected before final persistence design.

## Candidate X Actions

- Save Post
- Save Reply
- Save as Source Lead
- Add Note or Tags
- Mark for Possible Reply Draft

Thread capture is excluded from the first version. A human may save individual posts separately when needed.

## Failure Behavior

When extraction is uncertain or fails:

- do not guess
- do not save silently
- show the current URL
- permit manual selected-text or note entry
- record parser failure evidence

Parser failures must never become corrupted captures disguised as success.

## Publishing Separation

Capture and publishing are separate concerns.

The extension must not:

- post
- reply
- like
- follow
- repost
- send DMs
- press platform submission controls

X publishing remains manual copy/paste during the admitted phases.

Truth Social publishing remains manual official-UI or official share-composer use, outside the capture extension.

## Screenshots

Automatic screenshots are excluded.

Any screenshot workflow remains a separate, deliberate human action governed by platform-specific source, copyright, privacy, and retention rules.

## Security

- local backend only
- no platform credentials stored
- no session tokens or cookies read or reused
- least-privilege browser permissions
- no broad host access beyond the admitted X pages
- no background monitoring
- clear on/off and uninstall behavior

## Required Proof Before Admission

- current X policy review
- threat and privacy review
- proof no background collection occurs
- parser-failure tests
- malformed-page tests
- human-confirmation tests
- local authentication tests
- volume-drift safeguards
- owner approval of the final bounded implementation packet

## Recheck Rule

Revisit Truth Social capture only when one of these occurs:

- Truth Social publishes explicit browser-extension or developer guidance
- written permission is received
- professional legal review supports a narrow owned-content path
- the owner explicitly reopens the decision

Until then, no Truth Social parser belongs in the codebase. The drawer is closed.
