# Workflow Overview

## Basic Loop

```text
1. User captures a lead.
2. Lead enters inbox.
3. User or LLM classifies topic/source type.
4. Research notes and source summaries are added.
5. LLM drafts one or more post options.
6. Human reviews claims, tone, and risk.
7. Human approves exact final text.
8. User manually posts to X.
9. User records published URL.
10. Metrics are tracked later.
```

## Lead Types

- X post
- X reply
- article
- YouTube video
- podcast
- PDF/document
- archive/museum page
- academic paper
- old newspaper scan
- forum/thread
- manual idea
- bread/bureaucracy joke

## Source to Post Flow

For article-linked posts:

```text
source/article → post idea → draft → review → published X post → replies/comments → future leads
```

Important relationship:

```text
sources → posts → replies/comments → reply drafts/human reviews → metrics_snapshots
```

## Capture Before Automation

Manual capture and manual posting are features, not failures.

They keep the project safe and inspectable while the workflow is still young.
